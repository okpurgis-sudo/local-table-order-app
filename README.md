# テーブル注文アプリ

文化祭・模擬店・小規模イベント向けの、ローカルネットワーク内で動くテーブル注文アプリです。

インターネット上のサービスではなく、1台のWindows PCを注文サーバーとして起動し、同じネットワーク内のタブレットやスマホから注文画面を開いて使います。

## 主な機能

- テーブル別注文画面
- 管理画面
- 管理画面パスワード
- 管理PC専用のパスワードリセット
- 商品管理
- 商品ごとの最大注文数
- 商品ごとの売り切れ設定
- 商品カテゴリ
- 並び順
- おすすめ表示
- 表示 / 非表示
- 注文受付 ON / OFF
- テーブル使用中ロック
- 未提供注文の表示
- 注文履歴の閲覧
- 提供済み履歴の詳細表示
- テーブル状態の管理
- Windows exe化用ファイル
- Inno Setup インストーラー用ファイル

## 必要環境

### ソースから起動する場合

- Windows 10 / 11
- Python 3.10 以上推奨

### exe化する場合

- Python
- PyInstaller
- Inno Setup（インストーラーを作る場合）

## 起動方法

```bat
run_windows.bat
```

起動すると管理画面が自動で開きます。

管理画面:

```text
http://127.0.0.1:5000/admin
```

タブレット側は、起動時の黒い画面に表示されたURLを開きます。

例:

```text
http://192.168.x.x:5000/table/1
http://192.168.x.x:5000/table/2
```

## 基本運用

1. サーバーPCでアプリを起動する
2. 管理画面を開く
3. 初期パスワード `admin` でログインする
4. 注文受付を停止する
5. 必要に応じてアプリ設定から管理画面パスワードや表示名を変更する
6. 商品管理で商品を設定する
7. 注文受付を開始する
8. 各タブレットでテーブルURLを開く
9. 客が注文する
10. 管理画面で注文を確認する
11. 商品を渡したら「提供済みにする」を押す
12. 会計・片付け後、「使用可能にする」を押す


## 管理画面パスワード

管理画面を開くにはパスワードが必要です。

初期パスワード:

```text
admin
```

初期パスワードのまま使うのは避け、運用前に管理画面の「アプリ設定」から変更してください。
アプリ設定は、誤操作防止のため注文受付を停止しているときだけ開けます。

### パスワードを忘れた場合

管理PCでログイン画面下の「パスワードを忘れた場合（管理PCのみ）」を開くと、管理画面パスワードを初期値 `admin` に戻せます。

リセットを実行するには、確認欄に次の文字を入力します。

```text
RESET
```

内部URLは以下です。

```text
http://127.0.0.1:5000/admin/password-reset
```

このリセット画面は、サーバーPC自身から開いた場合のみ使えます。
タブレットや別端末からは実行できません。

## 注文履歴

管理画面の「注文履歴」から、未提供・提供済みを含むすべての注文を確認できます。

注文履歴では以下を確認できます。

- 注文番号
- テーブル番号
- 注文日時
- 注文状態
- 商品名
- 数量
- 小計
- 合計

状態やテーブル番号で簡単に絞り込むこともできます。

提供済みの注文履歴は、注文履歴画面から削除できます。
ただし、誤操作防止のため、履歴削除は注文受付を停止しているときだけ実行できます。
削除時は確認欄に次の文字を入力します。

```text
削除する
```

未提供の注文は削除対象にしないため、配膳中の注文管理は残ります。

## アプリ設定

アプリ設定では、以下を変更できます。

- アプリ表示名
- 画面アイコン
- 管理画面パスワード

アプリ設定は、注文受付を停止しているときだけ開けます。
営業中に表示名・アイコン・管理パスワードを変更してしまう誤操作を防ぐためです。

## 商品管理

商品管理は、注文受付を停止しているときだけ開けます。

設定できる項目:

- 商品名
- 内容・単位・説明
- 価格
- 最大注文数
- カテゴリ
- 並び順
- 注文画面に表示
- おすすめ表示
- 売り切れ

### 商品設定例

| 商品名 | 内容・単位・説明 | 価格 | 最大注文数 | カテゴリ |
|---|---|---:|---:|---|
| 焼きそば | 1パック | 500 | 5 | フード |
| ジュース | 1杯 | 200 | 10 | ドリンク |
| からあげ | 5個 | 400 | 5 | フード |

## テーブルロックについて

各テーブルURLを端末で開くと、その端末がそのテーブルに割り当てられます。

そのため、別の端末で同じテーブルを開こうとすると、使用中画面が表示されます。

管理画面の「ロック解除」は、タブレットを入れ替えるときや、誤って別の端末で開いたときだけ使ってください。

## インターネットなし運用

このアプリはローカルネットワーク内で動きます。

そのため、インターネット接続がなくても、以下のどちらかで使えます。

- ノートPCのモバイルホットスポットを使う
- Wi-Fiルーターを使う

必要なのは、サーバーPCとタブレットが同じネットワークに接続されていることです。

## exe化

```bat
build_exe.bat
```

成功すると、次の場所にexeが作成されます。

```text
dist\LocalTableOrderApp\LocalTableOrderApp.exe
```

## インストーラー化

Inno Setup で `installer.iss` を開き、Compileしてください。

成功すると、次の場所にインストーラーが作成されます。

```text
installer\LocalTableOrderApp_Setup.exe
```

## データファイルについて

注文履歴やテーブル状態は `data/` フォルダ内のJSONファイルに保存されます。

通常運用では、管理画面から注文履歴の閲覧やテーブル状態の操作を行ってください。
誤操作防止のため、管理画面には一括初期化ボタンを置いていません。

開発中やテスト時に手動で履歴を消す場合は、アプリを停止してから以下を編集します。

注文履歴だけ消す場合:

```text
data/orders.json -> []
```

テーブル状態とカートも消す場合:

```text
data/table_sessions.json -> {}
data/carts.json -> {}
```

## 注意

- 本番前に必ずテスト注文を行ってください。
- アプリ設定・商品管理・履歴削除は、注文受付を停止してから行ってください。
- 注文受付停止中は、注文開始・追加注文・注文確定はできません。
- Windowsファイアウォールで通信が止まる場合があります。
- タブレットから開けない場合は、サーバーPCと同じネットワークにいるか確認してください。
- ゲストWi-FiやAP隔離が有効なネットワークでは、端末同士の通信ができない場合があります。
